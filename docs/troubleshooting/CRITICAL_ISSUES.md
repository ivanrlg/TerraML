## Formato para Critical Issues

Cuando se identifique un critical issue, documentar asi:

### Issue #N: [Titulo]

- **Severity**: CRITICAL
- **Discovered**: YYYY-MM-DD
- **Status**: Activo / Resuelto

**Problema**: Que pasa.

**Causa raiz**: Por que pasa.

**Solucion/Workaround**:
```
Codigo o pasos
```

**Archivos afectados**:
- `ruta/archivo.cs`

**Testing checklist**:
- [ ] Verificar que el fix funciona
- [ ] Verificar que no introduce regresiones
